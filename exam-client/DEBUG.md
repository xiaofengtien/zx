# 调试和日志查看指南

## 查看应用日志

### 方法 1：开发者工具（推荐）

应用已经配置为自动打开开发者工具。如果看到空白页面：

1. **查看控制台**：在开发者工具中切换到 "Console" 标签
2. **查看错误信息**：红色错误信息会显示具体问题
3. **查看网络请求**：切换到 "Network" 标签查看 API 请求

### 方法 2：终端日志

在终端中运行应用（开发模式）：

```bash
cd exam-client
npm run dev
```

所有日志会输出到终端。

### 方法 3：查看打包后的应用日志

如果运行打包后的应用，可以通过以下方式查看日志：

#### macOS
```bash
# 打开应用后，在终端运行：
/Applications/考试客户端.app/Contents/MacOS/考试客户端

# 或者查看系统日志：
log show --predicate 'process == "考试客户端"' --last 5m
```

## 常见问题排查

### 1. 空白页面

**可能原因**：
- HTML 文件路径不正确
- JavaScript 加载失败
- 路由配置问题

**解决方法**：
1. 打开开发者工具（Cmd+Option+I）
2. 查看 Console 标签的错误信息
3. 查看 Network 标签，确认文件是否加载成功

### 2. 登录页面不显示

**检查项**：
- 确认 `src/renderer/router/index.js` 中路由配置正确
- 确认 `src/renderer/main.js` 中 Vue 实例正确挂载
- 查看 Console 是否有 Vue 相关错误

### 3. 数据库连接失败

**检查项**：
- 查看 Console 中的数据库初始化日志
- 确认应用数据目录权限正常
- 检查 SQLite 文件是否创建成功

## 调试技巧

### 1. 添加更多日志

在主进程文件 `.electron-vue/main.js` 中已经添加了日志输出：
- 应用启动信息
- 文件路径信息
- 错误信息

### 2. 检查文件路径

如果看到 "文件未找到" 错误，开发者工具会显示：
- 尝试的所有路径
- `__dirname` 的值
- `app.getAppPath()` 的值

### 3. 网络请求调试

如果登录失败：
1. 打开开发者工具 → Network 标签
2. 尝试登录
3. 查看 API 请求和响应
4. 检查请求 URL 是否正确
5. 检查响应状态码和内容

## 重新打包

如果修复了问题，需要重新打包：

```bash
# 1. 重新构建
npm run build

# 2. 重新打包
npm run pack:mac
```
