# 应用数据目录说明

## 应用名称
- **应用名称**：`择学考试端`
- **App ID**：`com.exam.client`

## 应用数据根目录

应用使用 Electron 的 `app.getPath('userData')` 来获取用户数据目录，该路径在不同操作系统下的位置如下：

### macOS
```
~/Library/Application Support/择学考试端
```

**完整路径示例**（假设用户名为 `Stephen`）：
```
/Users/Stephen/Library/Application Support/择学考试端
```

### Windows
```
%APPDATA%\择学考试端
```

**完整路径示例**（假设用户名为 `Administrator`）：
```
C:\Users\Administrator\AppData\Roaming\择学考试端
```

**注意**：在 Windows 系统中，`%APPDATA%` 环境变量指向 `C:\Users\[用户名]\AppData\Roaming` 目录。

## 如何手动放置 ZIP 文件

如果您需要手动将 ZIP 文件放置到本地以供应用程序使用（例如离线分发），请按照以下步骤操作：

### macOS
1. 打开 Finder
2. 按 `Cmd + Shift + G` 打开"前往文件夹"对话框
3. 输入路径：`~/Library/Application Support/择学考试端/paper_packages`
4. 将 ZIP 文件复制到此目录
5. 确保文件名格式为：`{paperCode}_v{version}.zip`

### Windows
1. 按 `Win + R` 打开"运行"对话框
2. 输入：`%APPDATA%\择学考试端\paper_packages`
3. 按回车键打开该目录
4. 将 ZIP 文件复制到此目录
5. 确保文件名格式为：`{paperCode}_v{version}.zip`

**或者**：
1. 打开文件资源管理器
2. 在地址栏输入：`C:\Users\[您的用户名]\AppData\Roaming\择学考试端\paper_packages`
3. 将 ZIP 文件复制到此目录

## 开发模式 vs 生产模式

### 开发模式（Development）
在开发模式下，Electron 可能会使用不同的应用名称。代码中已通过 `app.setName('择学考试端')` 确保路径一致。

### 生产模式（Production）
打包后的应用会使用 `package.json` 中配置的 `productName: "择学考试端"`，确保路径与开发模式一致。

## 查看实际路径

您可以通过以下方式在应用中查看实际的数据目录路径：

1. **通过 IPC 调用**：
   - 在渲染进程中调用：`window.electron.ipcRenderer.invoke('app:getUserDataPath')`
   - 返回值为完整的用户数据目录路径

2. **查看控制台日志**：
   - 应用启动时，会在主进程控制台输出数据库文件路径
   - 格式：`数据库文件路径: {完整路径}`

## 注意事项

1. **权限问题**：
   - macOS：确保应用有权限访问 `~/Library/Application Support` 目录
   - Windows：确保应用有权限访问 `%APPDATA%` 目录

2. **路径中的特殊字符**：
   - 应用名称包含中文字符，路径中会直接使用中文字符
   - 在命令行或脚本中使用时，注意处理中文字符编码

3. **备份建议**：
   - 建议定期备份 `exam.db` 数据库文件
   - 建议备份 `paper_packages` 目录中的 ZIP 文件

4. **清理建议**：
   - `temp` 目录中的临时文件可以定期清理
   - 旧版本的 ZIP 文件可以手动删除（应用会自动清理）
