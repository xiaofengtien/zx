# 阶段七：答题数据流设计（方案B确认）

## 一、数据来源确认

### 1. 试卷信息获取（方案B）
- **来源**：ZIP包解压后的 `manifest.json`
- **包含字段**：
  - `paperId` - 试卷ID（用于保存答题结果）
  - `paperCode` - 试卷编码
  - `paperName` - 试卷名称（用于显示）
  - `totalScore` - 总分
  - `totalQuestions` - 题目总数
  - `duration` - 考试时长
  - 其他试卷配置信息

### 2. 题目信息获取
- **来源**：ZIP包解压后的 `questions.json`
- **包含字段**：
  - `id` - 题目ID（用于保存答题结果）
  - `title` - 题目内容
  - `type` - 题目类型
  - `answers` - 答案选项列表
  - `blankAreas` - 完形填空区域（如果有）
  - 其他题目信息

## 二、答题数据存储流程

### 1. 开始答题
```
用户选择试卷类型
  ↓
根据试卷类型查询 paperCode（从SQLite的paper表，或从ZIP包列表）
  ↓
调用 paperService.extractPaperPackage(paperCode, version)
  ↓
获取 manifest.json 和 questions.json
  ↓
从 manifest 获取 paperId 和 paperName
  ↓
调用 answerService.startExam({
  paperId: manifest.paperId,
  paperName: manifest.paperName,
  appUserId: userInfo.userId
})
  ↓
保存到 SQLite: app_user_paper_info 表
  - paper_id: manifest.paperId
  - paper_name: manifest.paperName
  - app_user_id: userId
  - start_time: 当前时间戳
  - is_submit: 0
  - sync_status: 0
```

### 2. 答题过程中保存结果
```
用户答题
  ↓
调用 answerService.saveQuestionResult({
  paperId: manifest.paperId,        // 从manifest获取
  questionId: question.id,           // 从questions.json获取
  answerIds: "1,2",                  // 用户选择的答案ID
  userAnswer: "A,B",                 // 用户答案文本
  result: 1,                         // 0-错误，1-正确（需要判断）
  questionSort: index + 1,           // 题目序号
  timeSpent: 30                      // 用时（秒）
})
  ↓
保存到 SQLite: app_user_paper_question_result 表
  - paper_id: manifest.paperId
  - question_id: question.id
  - answer_ids: "1,2"
  - user_answer: "A,B"
  - result: 1
  - question_sort: 1
  - sync_status: 0（未同步）
```

### 3. 提交答题
```
用户点击提交
  ↓
计算总分、得分、正确数、错误数
  ↓
调用 answerService.submitExam({
  paperInfoId: paperInfoId,
  paperId: manifest.paperId,
  totalScore: manifest.totalScore,
  userScore: calculatedScore,
  correctCount: correctCount,
  wrongCount: wrongCount
})
  ↓
更新 SQLite: app_user_paper_info 表
  - is_submit: 1
  - submit_time: 当前时间戳
  - used_time: 用时（秒）
  - total_score, user_score, correct_count, wrong_count
  - sync_status: 0（未同步）
```

## 三、同步到MySQL流程

### 1. 在线同步时机
- 提交答题后立即同步
- 应用启动时同步所有未同步的记录
- 手动触发同步

### 2. 同步数据格式
```javascript
// 从SQLite读取答题结果
const paperInfo = await answerService.getPaperInfo(paperInfoId)

// 构建提交数据（符合服务端SubmitPaperBO格式）
const submitData = {
  businessType: 5,                    // 考试业务类型
  businessId: paperInfo.paper_id,     // 试卷ID（从manifest.paperId获取）
  appUserId: paperInfo.app_user_id,
  questionResults: paperInfo.questionResults.map(r => ({
    questionId: r.question_id,         // 从questions.json获取
    answerIds: r.answer_ids,
    userAnswer: r.user_answer,
    result: r.result
  })),
  blankResults: paperInfo.blankResults.map(r => ({
    questionId: r.question_id,         // 从questions.json获取
    blankAreaId: r.blank_area_id,
    blankIndex: r.blank_index,
    answerIds: r.answer_ids,
    result: r.result
  }))
}

// 调用服务端接口
POST /student/paper/submit
```

### 3. 同步状态管理
- **sync_status = 0**：未同步（默认）
- **sync_status = 1**：已同步
- 同步成功后更新所有相关记录的 sync_status = 1

## 四、SQLite表结构确认

### 1. app_user_paper_info（答题记录主表）
```sql
CREATE TABLE app_user_paper_info (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  paper_id INTEGER NOT NULL,          -- 从manifest.paperId获取
  paper_name TEXT,                    -- 从manifest.paperName获取
  app_user_id INTEGER NOT NULL,
  start_time INTEGER,
  submit_time INTEGER,
  used_time INTEGER,
  total_score REAL DEFAULT 0,
  user_score REAL DEFAULT 0,
  correct_count INTEGER DEFAULT 0,
  wrong_count INTEGER DEFAULT 0,
  is_submit INTEGER DEFAULT 0,
  sync_status INTEGER DEFAULT 0,      -- 0-未同步，1-已同步
  sync_time INTEGER,
  create_time INTEGER,
  update_time INTEGER
)
```

### 2. app_user_paper_question_result（题目答题结果表）
```sql
CREATE TABLE app_user_paper_question_result (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  paper_id INTEGER NOT NULL,          -- 从manifest.paperId获取

  answer_ids TEXT,
  user_answer TEXT,

  answer_time INTEGER,
  time_spent INTEGER,
  is_reviewed INTEGER DEFAULT 0,
  sync_status INTEGER DEFAULT 0,      -- 0-未同步，1-已同步
  create_time INTEGER
)
```

### 3. app_user_paper_question_blank_result（完形填空结果表）

```sql
CREATE TABLE app_user_paper_question_blank_result (

  blank_area_id INTEGER,
  blank_index INTEGER,
  answer_ids TEXT,

  sync_status INTEGER DEFAULT 0,      -- 0-未同步，1-已同步

)
```

## 五、关键确认点

### ✅ 确认1：数据来源
- **试卷信息**：从 `manifest.json` 获取（包含 paperId, paperName 等）
- **题目信息**：从 `questions.json` 获取（包含 questionId 等）
- **不需要**：从SQLite的paper表查询试卷信息

### ✅ 确认2：答题结果存储
- **必须存储到SQLite**：支持离线答题
- **存储字段**：
  - `paper_id`：从 `manifest.paperId` 获取
  - `question_id`：从 `questions.json` 的 `id` 字段获取
  - 其他答题数据

### ✅ 确认3：同步到MySQL
- **同步时机**：在线时自动同步，离线时缓存到SQLite
- **同步数据**：从SQLite读取答题结果，构建SubmitPaperBO格式
- **关键字段**：
  - `businessId`：使用 `paper_id`（从manifest.paperId获取）
  - `questionId`：使用 `question_id`（从questions.json获取）
  - 这些ID都是MySQL中的真实ID，可以直接使用

### ✅ 确认4：数据一致性
- **paperId**：manifest.json中的paperId = MySQL的paper.id
- **questionId**：questions.json中的id = MySQL的question.id
- **同步时**：直接使用这些ID，无需额外映射

## 七、总结

### 方案B的优势
1. ✅ **数据最新**：直接从ZIP包获取，包含所有新增字段
2. ✅ **无需同步**：不需要将试卷信息同步到SQLite的paper表
3. ✅ **ID完整**：manifest和questions都包含MySQL的真实ID
4. ✅ **离线支持**：答题结果保存到SQLite，支持离线答题

### 关键点
- **paperId** 和 **questionId** 都来自ZIP包的JSON数据
- 这些ID是MySQL中的真实ID，可以直接用于同步
- 答题结果存储到SQLite，同步时从SQLite读取并提交

### 确认
✅ **可以使用方案B**：直接从ZIP解压后的JSON数据获取试卷和题目信息
✅ **答题结果存储到SQLite**：支持离线答题
✅ **同步到MySQL**：在线时从SQLite读取并提交，使用manifest和questions中的真实ID
