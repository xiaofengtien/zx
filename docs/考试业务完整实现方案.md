## 概述

### 设计目标

1. **试卷管理**：支持后台创建、编辑、删除试卷
2. **题目关联**：试卷与题目通过关联表建立关系
3. **媒体文件**：题目媒体文件直接关联，无需通过AppMaterial
4. **答题流程**：先生成试卷 → 查询题目 → 答题 → 同步结果
5. **离线支持**：SQLite存储试卷包（ZIP格式），支持离线答题

### 核心变更

- 新增 `paper` 表（试卷主表）
- 新增 `paper_question` 表（试卷-题目关联表）
- 新增 `question_media` 表（题目媒体文件表）
- 调整 `app_user_paper_info` 表（增加paper_id字段）
- 调整 `question_category_business_settings` 表（初始化数据，默认类型5）

## 媒体文件存储架构

### 存储策略

#### 后台存储（服务端）

**方案A：OSS存储（推荐，生产环境）**
- 媒体文件上传到OSS
- ZIP包上传到OSS
- 数据库记录OSS URL/路径

**方案B：服务器本地存储（开发/小规模）**
- 媒体文件保存到服务器本地文件系统
- ZIP包保存到服务器本地文件系统
- 通过Nginx等提供HTTP访问

#### 客户端存储（离线端）

**必须：本地文件存储**
- 从ZIP包解压媒体文件到本地
- SQLite存储本地相对路径
- 答题时从本地路径加载

### 存储结构

```
应用数据目录/
├── exam.db (SQLite数据库)
├── paper_packages/          # 试卷包存储目录
│   ├── {paper_code}_v{version}.zip
│   └── {paper_code}_v{version}.zip.part1 (分片)
└── media/                    # 媒体文件存储目录（解压后）
    ├── {paper_code}/         # 按试卷分组
    │   ├── q_{question_id}/  # 按题目分组
    │   │   ├── title.jpg     # 题目媒体
    │   │   └── options/      # 选项媒体
    │   │       ├── a.mp3
    │   │       └── b.mp3
    │   └── q_{question_id}/
    │       └── video.mp4
    └── intro/                 # 试卷独白音频
        └── {paper_code}.mp3
```

### 文件大小阈值

| 文件大小 | 存储方式 | 说明 |
|---------|---------|------|
| < 50MB | SQLite BLOB | 直接存储在数据库中 |
| 50-200MB | 文件系统 | 存储在文件系统，SQLite存储路径 |
| > 200MB | 分片下载 + 文件系统 | 支持HTTP Range断点续传，分片大小10MB |

## 同步机制

### 同步策略

#### 启动时同步

1. **检查版本**：比较本地版本号与服务器版本号
2. **全量/增量同步**：
   - 首次启动：全量同步所有试卷包
   - 后续启动：检查 `package_hash`，不一致时增量同步

#### 联网时同步

1. **定期增量同步**：5分钟轮询检查更新
2. **手动触发**：支持手动强制全量同步
3. **断点续传**：支持HTTP Range请求，分片下载

### 同步流程

```
启动时：
1. 检查本地是否有数据
   - 无数据 → 全量同步所有试卷包
   - 有数据 → 发送本地版本信息 → 增量同步

联网时：
1. 定期轮询（5分钟）或手动触发
2. 发送本地版本信息
3. 下载更新的试卷包（支持断点续传）
```

## 缓存策略

### 解压结果缓存

- **策略**：LRU（最近最少使用）
- **大小限制**：最大100MB
- **清理机制**：超过限制时自动清理最久未使用的缓存

### 媒体文件按需加载

- 答题时按需加载媒体文件
- 不预加载所有媒体文件，节省内存

## 版本管理

### 版本保留策略

- **保留版本数**：当前版本 + 最近3个版本
- **清理周期**：90天
- **清理前检查**：检查是否有用户在使用该版本

### 清理流程

```javascript
/**
 * 清理旧版本试卷包和媒体文件
 */
async cleanupOldVersions(retentionDays = 90) {
  const cutoffTime = Date.now() - (retentionDays * 24 * 60 * 60 * 1000)
  
  // 1. 查询需要清理的试卷包（保留当前版本+最近3个版本）
  const oldPackages = this.db.prepare(`
    SELECT paper_code, version FROM paper_package
    WHERE sync_time < ? AND is_active = 0
    AND version NOT IN (
      SELECT version FROM paper_package 
      WHERE paper_code = paper_package.paper_code 
      ORDER BY version DESC LIMIT 4
    )
  `).all(cutoffTime)
  
  // 2. 检查是否有用户在使用（通过app_user_paper_info表）
  for (const pkg of oldPackages) {
    const inUse = this.db.prepare(`
      SELECT COUNT(*) FROM app_user_paper_info 
      WHERE paper_id = ? AND is_submit = 0
    `).get(pkg.paper_id)
    
    if (inUse > 0) {
      continue // 跳过正在使用的版本
    }
    
    // 3. 删除旧版本媒体文件
    const mediaDir = path.join(this.mediaBasePath, pkg.paper_code)
    if (fs.existsSync(mediaDir)) {
      fs.rmSync(mediaDir, { recursive: true, force: true })
    }
    
    // 4. 删除旧版本ZIP包
    if (pkg.package_path && fs.existsSync(pkg.package_path)) {
      fs.unlinkSync(pkg.package_path)
    }
  }
  
  // 5. 从数据库删除记录
  this.db.prepare(`
    DELETE FROM paper_package WHERE sync_time < ? AND is_active = 0
  `).run(cutoffTime)
}
```

### 3. 题库分类递归算法优化

**问题**：数量大的情况下递归算法存在性能问题

**优化方案**：

#### 方案A：使用MySQL递归CTE（推荐）

```sql
        FROM question_category

        -- 递归查询子节点

    SELECT * FROM category_tree
```

#### 方案B：一次性查询所有数据，在内存中构建树（适合数据量不大）

```java
@Override
public List<QuestionCategoryDTO> getCategoryTree(QuestionCategoryQueryBO queryBO) {
    // 1. 一次性查询所有分类（避免N+1查询）
    LambdaQueryWrapper<QuestionCategory> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Objects.nonNull(queryBO.getStatus()), QuestionCategory::getStatus, queryBO.getStatus())
          .like(StringUtils.isNotEmpty(queryBO.getName()), QuestionCategory::getName, queryBO.getName())
          .orderByAsc(QuestionCategory::getSortNum);
    
    List<QuestionCategory> allCategories = list(wrapper);
    
    if (CollectionUtils.isEmpty(allCategories)) {
        return new ArrayList<>();
    }
    
    // 2. 构建ID到分类的映射（O(1)查找）
    Map<Integer, QuestionCategoryDTO> categoryMap = allCategories.stream()
        .map(this::convertToDTO)
        .collect(Collectors.toMap(QuestionCategoryDTO::getId, Function.identity()));
    
    // 3. 构建树结构（单次遍历）
    List<QuestionCategoryDTO> rootList = new ArrayList<>();
    for (QuestionCategoryDTO category : categoryMap.values()) {
        Integer fatherId = category.getFatherId();
        if (fatherId == null || fatherId == 0) {
            rootList.add(category);
        } else {
            QuestionCategoryDTO parent = categoryMap.get(fatherId);
            if (parent != null) {
                if (parent.getChildren() == null) {
                    parent.setChildren(new ArrayList<>());
                }
                parent.getChildren().add(category);
            }
        }
    }
    
    // 4. 设置统计信息（批量查询，避免N+1）
    setCategoryStats(rootList, categoryMap.keySet());
    
    return rootList;
}

/**
 * 批量设置分类统计信息（优化：避免N+1查询）
 */
private void setCategoryStats(List<QuestionCategoryDTO> categories, Set<Integer> categoryIds) {
    // 批量查询题目数量
    Map<Integer, Integer> questionCountMap = questionMapper.countByCategoryIds(categoryIds);
    
    // 递归设置统计信息
    for (QuestionCategoryDTO category : categories) {
        category.setQuestionCount(questionCountMap.getOrDefault(category.getId(), 0));
        if (category.getChildren() != null && !category.getChildren().isEmpty()) {
            setCategoryStats(category.getChildren(), categoryIds);
        }
    }
}
```

**优化要点**：
1. ✅ 使用递归CTE或一次性查询，避免N+1查询
2. ✅ 使用Map进行O(1)查找，避免重复遍历
3. ✅ 批量查询统计信息，避免逐条查询
4. ✅ 限制递归深度，防止无限递归

```javascript
    console.log('执行全量覆盖：删除并重新创建试卷相关表...')
    
    try {
      // 1. 删除现有表（如果存在）
      for (const tableName of toCreate) {
        try {
          this.db.exec(`DROP TABLE IF EXISTS ${tableName}`)
          console.log(`  ✓ 删除表: ${tableName}`)
        } catch (error) {
          console.warn(`  删除表 ${tableName} 失败:`, error.message)
        }
      }
      
      // 2. 删除相关索引
      const indexes = [
        'idx_paper_package_code',
        'idx_paper_index_code',
        'idx_paper_info_user',
        'idx_paper_info_sync',
        'idx_paper_result_user',
        'idx_paper_result_sync',
        'idx_paper_blank_sync',
        'idx_paper_question'
      ]
      
      for (const indexName of indexes) {
        try {
          this.db.exec(`DROP INDEX IF EXISTS ${indexName}`)
        } catch (error) {
          // 索引可能不存在，忽略错误
        }
      }
      
      // 3. 重新创建表
      this.initPaperTables(toCreate)
      
      // 4. 重新创建索引
      this.createPaperIndexes()
      
      console.log('✓ 全量覆盖完成：试卷相关表已重新创建')
      
    } catch (error) {
      console.error('✗ 全量覆盖失败:', error)
      // 即使全量覆盖失败，也不抛出异常（避免影响应用启动）
    }
  }

  /**
   * 创建关键表索引（同步）
   */
  createCriticalIndexes() {
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_student_account ON student_credentials(student_account)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_student_papers_account ON student_papers(student_account)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_dict_type ON dict_data(dict_type)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_student_archive_account ON student_archive(student_account)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_student_archive_user_id ON student_archive(user_id)`)
  }

  /**
   * 创建试卷相关表索引（异步）
   */
  createPaperIndexes() {
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_package_code ON paper_package(paper_code)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_index_code ON paper_index(paper_code)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_info_user ON app_user_paper_info(app_user_id)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_info_sync ON app_user_paper_info(sync_status)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_result_user ON app_user_paper_question_result(paper_id, app_user_id)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_result_sync ON app_user_paper_question_result(sync_status)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_blank_sync ON app_user_paper_question_blank_result(sync_status)`)
    this.db.exec(`CREATE INDEX IF NOT EXISTS idx_paper_question ON question_media_index(paper_id, question_id)`)
  }

  /**
   * 检查表是否存在（用于首次使用前检查）
   * 如果表不存在，尝试创建
   */
  ensurePaperTablesExist() {
    const paperTables = [
      'paper_package',
      'paper_index',
      'app_user_paper_info',
      'app_user_paper_question_result',
      'app_user_paper_question_blank_result',
      'question_media_index'
    ]
    
    const existingTables = this.getExistingTables()
    const missingTables = paperTables.filter(t => !existingTables.includes(t))
    
    if (missingTables.length > 0) {
      console.log(`检测到缺失的试卷相关表，立即创建: ${missingTables.join(', ')}`)
      try {
        this.initPaperTables(missingTables)
        this.createPaperIndexes()
        console.log('✓ 试卷相关表创建成功')
      } catch (error) {
        console.error('创建试卷相关表失败:', error)
        throw error
      }
    }
  }

  getDB() {
    return this.db
  }

  close() {
    if (this.db) {
      this.db.close()
    }
  }
}
```
