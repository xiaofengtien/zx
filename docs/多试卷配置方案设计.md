# 多试卷配置方案设计

## 一、需求确认

### 用户需求
1. **学员档案配置方式**：
   - 在后台通过弹窗选择具体试卷，支持多选
   - 存储多个试卷ID，用逗号分割（或JSON数组）

2. **同步逻辑调整**：
   - **当前**：只同步相同（年月+省+类型）组合的最新版本
   - **新需求**：根据学员档案配置的试卷ID列表，同步指定的多个试卷
   - 支持同一学员配置多个试卷（可以是同一份试卷的不同版本，或不同试卷）

3. **灵活性**：
   - 完全由管理员在后台配置
   - 学员登录后，自动同步配置的试卷
   - 客户端无需选择试卷

## 二、当前同步逻辑分析

### 当前实现（syncZipPackages）

```javascript
// 当前逻辑：按 (year, month, province, paper_type) 分组，只选择最新版本
const paperGroups = new Map()

for (const paper of papers) {
  const groupKey = `${year}_${month}_${province}_${paperType}`
  
  if (!paperGroups.has(groupKey)) {
    paperGroups.set(groupKey, paper)
  } else {
    // 比较版本，选择版本更大的
    if (version > existingVersion) {
      paperGroups.set(groupKey, paper)
    }
  }
}

// 只同步筛选后的试卷（每个组合只保留最新版本）
papers = Array.from(paperGroups.values())
```

### 问题
- ❌ 只同步最新版本，无法同步历史版本
- ❌ 无法同步同一组合的多个试卷（如：模拟一、模拟二、模拟三）
- ❌ 无法为不同学员配置不同试卷

## 三、新方案设计

### 方案：基于学员档案配置的试卷ID列表同步

#### 1. 数据库调整

```sql
-- 在学员档案表中增加字段
ALTER TABLE student_archive 
ADD COLUMN applicable_paper_ids VARCHAR(500) COMMENT '适用试卷ID列表（多个用逗号分隔，如：1,2,3）';

-- 或者使用JSON格式（更灵活）
ALTER TABLE student_archive 
ADD COLUMN applicable_paper_ids TEXT COMMENT '适用试卷ID列表（JSON数组，如：[1,2,3]）';
```

#### 2. 后台配置界面

**学员档案编辑页面**：
- 保留 `applicable_papers`（试卷类型）字段（用于权限控制）
- 新增 `applicable_paper_ids`（具体试卷ID）字段
- 点击"选择试卷"按钮，弹出试卷选择对话框
- 支持多选，显示试卷信息（名称、年月、省份、类型、版本等）
- **显示试卷包生成状态**：是否已生成试卷包、版本号、生成时间
- **支持操作生成试卷包**：列表中可以直接点击"生成试卷包"按钮
- 保存时，将选中的试卷ID列表存储到 `applicable_paper_ids`

#### 3. 同步逻辑调整

**新的同步流程**：

```javascript
// 1. 获取学员档案
const archive = await getStudentArchive(appUserId)

// 2. 解析配置的试卷ID列表
const paperIds = parsePaperIds(archive.applicablePaperIds) // [1, 2, 3]

// 3. 根据试卷ID列表查询试卷信息
const papers = await queryPapersByIds(paperIds, token)

// 4. 同步指定的试卷（不进行分组去重）
for (const paper of papers) {
  await syncPaperPackage(paper.paperCode, token)
}
```

**关键变化**：
- ✅ **不再按组合分组**：直接同步配置的试卷ID对应的试卷
- ✅ **支持多个试卷**：可以同步多个试卷（不同版本、不同试卷）
- ✅ **精确控制**：完全由管理员配置，客户端无需选择

#### 4. 客户端逻辑

**登录后同步**：
```javascript
// 1. 获取学员档案
const archive = await getStudentArchive(appUserId)

// 2. 解析试卷ID列表
const paperIds = JSON.parse(archive.applicablePaperIds || '[]')

// 3. 调用同步接口，同步指定的试卷
await syncService.syncPapersByIds(paperIds, token)
```

**答题时选择试卷**：
```javascript
// 1. 获取学员档案
const archive = await getStudentArchive(appUserId)

// 2. 解析试卷ID列表
const paperIds = JSON.parse(archive.applicablePaperIds || '[]')

// 3. 如果只有一个试卷，直接使用
if (paperIds.length === 1) {
  const paper = await getPaperById(paperIds[0])
  // 开始答题
}

// 4. 如果有多个试卷，显示选择列表（可选，或默认使用第一个）
// 或者：根据试卷类型自动匹配
```

## 四、实现细节

### 1. 数据库字段设计

**方案A：逗号分隔字符串**
```sql
applicable_paper_ids VARCHAR(500) -- "1,2,3"
```
- 优点：简单，易于查询
- 缺点：不便于扩展

**方案B：JSON数组（推荐）**
```sql
applicable_paper_ids TEXT -- "[1,2,3]"
```
- 优点：灵活，易于扩展（可以存储更多信息）
- 缺点：需要JSON解析

**推荐使用方案B**，因为：
- 可以扩展存储更多信息（如：试卷名称、版本等）
- 便于后续功能扩展

### 2. 同步接口调整

**新增接口**：根据试卷ID列表同步
```javascript
// syncService.js
async syncPapersByIds(paperIds, token) {
  // 1. 根据试卷ID列表查询试卷信息
  const papers = await this.queryPapersByIds(paperIds, token)
  
  // 2. 同步每个试卷的ZIP包
  for (const paper of papers) {
    await this.paperService.syncPaperPackage(paper.paperCode, token)
  }
}
```

**修改现有接口**：`syncZipPackages`
```javascript
// 方案1：保留原有逻辑，新增基于ID的同步方法
// 方案2：修改原有逻辑，根据学员档案配置决定同步方式
```

### 3. 后台配置界面

**试卷选择对话框**：
- 显示所有可用试卷列表
- 支持按试卷类型、年月、省份筛选
- 显示试卷信息：
  - 基本信息：名称、年月、省份、类型、版本、状态
  - **试卷包状态**：是否已生成试卷包（`package_hash` 是否为空）
  - **试卷包信息**：版本号、生成时间、包大小（如果有）
- 支持多选
- 已选中的试卷高亮显示
- **操作列**：
  - 如果未生成试卷包：显示"生成试卷包"按钮
  - 如果已生成试卷包：显示"重新生成"按钮（可选）
  - 点击后调用生成接口，显示加载状态，生成完成后刷新列表

**试卷包生成操作**：
```javascript
// 生成试卷包
async handleGeneratePackage(paperId) {
  this.$modal.confirm('是否确认生成试卷包？生成过程可能需要一些时间，请耐心等待。').then(() => {
    this.generatingPaperId = paperId
    generatePaperPackage({ id: paperId }).then(response => {
      this.$modal.msgSuccess("试卷包生成成功")
      this.refreshPaperList() // 刷新列表，更新试卷包状态
      this.generatingPaperId = null
    }).catch(() => {
      this.generatingPaperId = null
    })
  })
}
```

**保存逻辑**：
```javascript
// 保存时，将选中的试卷ID列表转为JSON数组
form.applicablePaperIds = JSON.stringify(selectedPaperIds) // "[1,2,3]"
```

### 4. 兼容性处理

**向后兼容**：
- 如果 `applicable_paper_ids` 为空，使用原有逻辑（按类型同步最新版本）
- 如果 `applicable_paper_ids` 有值，使用新逻辑（按ID列表同步）

```javascript
async syncZipPackages(token) {
  const archive = await getStudentArchive(appUserId)
  
  if (archive.applicablePaperIds && archive.applicablePaperIds.length > 0) {
    // 新逻辑：按ID列表同步
    await syncPapersByIds(archive.applicablePaperIds, token)
  } else {
    // 旧逻辑：按类型同步最新版本
    await syncZipPackagesByType(archive.applicablePapers, token)
  }
}
```

## 九、试卷包生成状态显示和操作

### 1. 试卷包状态判断

**判断标准**：
- 如果 `package_hash` 不为空 → 已生成试卷包
- 如果 `package_hash` 为空 → 未生成试卷包

**显示信息**：
- **已生成**：显示版本号、生成时间、包大小（如果有）
- **未生成**：显示"未生成"标签，并提供"生成试卷包"按钮

### 2. 试卷选择对话框界面设计

**表格列设计**：
```
| 选择 | 试卷名称 | 年月 | 省份 | 类型 | 版本 | 试卷包状态 | 操作 |
|------|---------|------|------|------|------|-----------|------|
| ☑    | 2023年12月河北中考英语听力模拟一 | 2023-12 | 河北 | 中考英语听力 | v1 | ✅ 已生成 (v1, 2024-01-15) | 重新生成 |
| ☐    | 2023年12月河北中考英语听力模拟二 | 2023-12 | 河北 | 中考英语听力 | v1 | ❌ 未生成 | 生成试卷包 |
```

**状态显示**：
- ✅ 已生成：绿色标签，显示版本号和生成时间
- ❌ 未生成：红色标签，显示"未生成"

**操作按钮**：
- 未生成：显示"生成试卷包"按钮（蓝色）
- 已生成：显示"重新生成"按钮（灰色，可选）

### 3. 生成试卷包操作流程

```javascript
// 1. 点击"生成试卷包"按钮
handleGeneratePackage(paperId) {
  // 2. 确认对话框
  this.$modal.confirm('是否确认生成试卷包？生成过程可能需要一些时间，请耐心等待。').then(() => {
    // 3. 显示加载状态
    this.generatingPaperId = paperId
    this.generatingStatus[paperId] = 'generating'
    
    // 4. 调用生成接口
    generatePaperPackage({ id: paperId }).then(response => {
      // 5. 生成成功
      this.$modal.msgSuccess("试卷包生成成功")
      this.refreshPaperList() // 刷新列表，更新状态
      this.generatingPaperId = null
      this.generatingStatus[paperId] = 'success'
    }).catch(error => {
      // 6. 生成失败
      this.$modal.msgError("试卷包生成失败：" + error.message)
      this.generatingPaperId = null
      this.generatingStatus[paperId] = 'failed'
    })
  })
}
```
