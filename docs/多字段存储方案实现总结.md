# 多字段存储方案实现总结

## 概述

从单个字段（`paper_code`）存储方案改为多个字段（`year`, `month`, `province`, `paper_type`, `custom_name`）存储方案，提高数据稳定性和可维护性。

## 实现内容

### 1. 数据库结构变更

#### MySQL DDL (`sql/add_paper_fields.sql`)

- 添加字段：
  - `year` INT(4) - 年份（2000-2050）
  - `month` INT(2) - 月份（1-12，可选）
  - `province` VARCHAR(50) - 省份编码（字典值：paper_province）
  - `custom_name` VARCHAR(100) - 自定义名称（可选）
- 添加唯一索引：`uk_paper_year_month_province_type` (`year`, `month`, `province`, `paper_type`)
  - 注意：如果 `month` 为 NULL，唯一性约束不适用（MySQL 的 NULL 值在唯一索引中视为不同值）

#### SQLite 迁移脚本 (`exam-client/src/database/migrations/002_add_paper_fields.js`)

- 添加字段：`year`, `month`, `province`, `custom_name`
- 创建唯一索引：`idx_paper_year_month_province_type` (`year`, `month`, `province`, `paper_type`)

### 2. 后端代码修改

#### 实体类和 DTO

- **Paper.java**: 添加 `year`, `month`, `province`, `custom_name` 字段
- **PaperDTO.java**: 添加相同字段
- **PaperCreateBO.java**: 已有这些字段（无需修改）
- **PaperUpdateBO.java**: 已有这些字段（无需修改）

#### Mapper 配置

- **PaperMapper.xml**: 
  - 在 `resultMap` 中添加字段映射
  - 在 `selectPaperVo` SQL 中添加字段查询

#### Service 层逻辑

- **PaperServiceImpl.java**:
  - **唯一性检查**：使用 `(year, month, province, paper_type)` 组合进行唯一性检查
    - 如果 `month` 为 null，则只检查 `(year, province, paper_type)`
    - 添加了 `getByYearMonthProvinceType` 方法用于唯一性检查
  - **paper_name 生成**：`generatePaperName` 方法只包含有值的字段
    - 如果 `month` 为 null，则不包含月份部分
    - 避免出现 "2025null河北中考模拟" 这种情况
  - **paper_code 生成**：保留用于向后兼容，但不再作为唯一性标识
  - **OSS 文件路径**：`buildPackageFileName` 方法使用 `paper_name` 作为文件名
    - 格式：`paper_packages/{year}{month}_{province}_{paper_type}/{paper_name}_v{version}.zip`
    - 如果 `month` 为 null，格式：`paper_packages/{year}_{province}_{paper_type}/{paper_name}_v{version}.zip`
  - **月份验证**：`month` 字段改为可选（不再强制要求）

#### Mapper 接口

- **PaperMapper.java**: 添加 `selectByYearMonthProvinceType` 方法
- **IPaperBiz.java**: 添加 `getByYearMonthProvinceType` 方法
- **PaperBizImpl.java**: 实现 `getByYearMonthProvinceType` 方法

### 3. 前端代码修改

#### edit.vue

- **移除 `parsePaperCode` 方法**：不再需要从 `paper_code` 解析字段
- **修改 `loadPaperData` 方法**：直接使用后端返回的 `year`, `month`, `province`, `customName` 字段
  - 使用 `Object.assign(this.form, response.data)` 直接赋值
  - 不再需要解析逻辑

#### add.vue

- 无需修改（已经直接使用字段）

### 4. 客户端同步逻辑

#### paperService.js

- **hash 检查**：在 `syncPaperPackage` 方法中，每次同步时：
  1. 强制从服务器获取最新的试卷信息（`getPaperInfo(paperCode, token, true)`）
  2. 检查本地 ZIP 包的 hash 是否与服务器一致
  3. 如果不一致，则下载新的 ZIP 包
- **离线支持**：如果网络请求失败，降级到使用本地缓存

## 关键特性

### 1. 唯一性约束

- **有月份时**：`(year, month, province, paper_type)` 组合唯一
- **无月份时**：`(year, province, paper_type)` 组合唯一（通过代码逻辑检查，因为 MySQL 唯一索引对 NULL 值的处理）

### 2. paper_name 生成规则

- 只包含有值的字段
- 格式：`{year}年{month}月{province}{paper_type_label}{custom}`
- 如果 `month` 为 null，格式：`{year}年{province}{paper_type_label}{custom}`
- 避免出现 "2025null河北中考模拟" 这种情况

### 3. OSS 文件路径

- 使用 `paper_name` 作为文件名（清理特殊字符后）
- 分组路径：`paper_packages/{year}{month}_{province}_{paper_type}/`
- 如果 `month` 为 null，分组路径：`paper_packages/{year}_{province}_{paper_type}/`

### 4. 客户端同步

- 每次联网同步时，强制从服务器获取最新的 hash
- 如果本地 hash 与服务器 hash 不一致，自动下载新的 ZIP 包
- 支持离线模式（使用本地缓存的 hash）

## 使用说明

### 1. 执行数据库迁移

#### MySQL

```sql
-- 执行 sql/add_paper_fields.sql
source sql/add_paper_fields.sql;
```

#### SQLite

迁移脚本会在客户端启动时自动执行（通过 `migration.js`）。

### 2. 前端使用

- **添加试卷**：填写年份、月份（可选）、省份、试卷类型、自定义名称（可选）
- **编辑试卷**：直接编辑字段，无需解析 `paper_code`

### 3. 注意事项

- **月份字段**：如果不需要月份，可以留空（null）
- **唯一性**：同一 `(year, month, province, paper_type)` 组合只能有一个试卷（除非使用 `custom_name` 区分）
- **向后兼容**：`paper_code` 字段仍然保留，用于向后兼容，但不作为唯一性标识
