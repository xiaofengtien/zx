# 100MB ZIP上传性能分析与优化方案

## 一、问题分析

### 1.1 当前实现问题

#### 问题1：100MB文件使用普通上传
- **当前逻辑**：`if (fileSize < 100L * 1024 * 1024)` 使用普通上传
- **问题**：100MB正好等于阈值，会使用普通上传，无法利用分片上传的并发优势
- **影响**：单线程上传，速度受限于单连接带宽

#### 问题2：七牛云并发数偏低
- **当前配置**：`resumableUploadMaxConcurrentTaskCount = 2`
- **问题**：只有2个并发，无法充分利用带宽
- **影响**：上传速度受限

#### 问题3：阿里云分片上传串行执行
- **当前实现**：使用while循环串行上传每个分片
- **问题**：10个分片（100MB/10MB）串行上传，总时间 = 10 × 单分片时间
- **影响**：无法利用并发优势，上传时间过长

#### 问题4：分片大小可能不是最优
- **当前配置**：10MB分片
- **问题**：100MB文件产生10个分片，分片数量多，网络请求开销大
- **影响**：每个分片都需要建立连接，增加延迟

### 1.2 OSS服务商带宽限制

#### 七牛云OSS
- **免费版/标准版**：未公开具体带宽限制
- **可能限制**：
  - 免费版可能有带宽限制（通常较低）
  - 标准版带宽更高，但具体数值需咨询客服
- **建议**：联系七牛云客服确认当前套餐的带宽限制

#### 阿里云OSS
- **外网上传带宽**：
  - 华东2（上海）、华南1（深圳）、华北2（北京）：10 Gbps
  - 华东1（杭州）：20 Gbps
  - 其他地域：10 Gbps
- **实际速度**：受账号等级、地域、网络环境影响
- **流控**：达到带宽阈值时会被流控，返回`x-oss-qos-delay-time`头

### 1.3 网络环境因素

- **服务器到OSS的网络质量**：延迟、丢包率
- **服务器带宽**：服务器出口带宽限制
- **地域选择**：服务器与OSS地域距离

## 二、优化方案

### 2.1 代码优化（推荐优先级：高）

#### 优化1：调整上传策略阈值
```java
// 修改前：fileSize < 100MB 使用普通上传
// 修改后：fileSize <= 50MB 使用普通上传，> 50MB 使用分片上传
if (fileSize <= 50L * 1024 * 1024) {
    // 普通上传
} else {
    // 分片上传
}
```
**预期效果**：100MB文件使用分片上传，可并发上传，速度提升2-4倍

#### 优化2：增加七牛云并发数
```java
// 修改前：cfg.resumableUploadMaxConcurrentTaskCount = 2
// 修改后：cfg.resumableUploadMaxConcurrentTaskCount = 4 或 8
cfg.resumableUploadMaxConcurrentTaskCount = 4; // 建议4，可根据实际情况调整到8
```
**预期效果**：并发数增加，上传速度提升50%-100%

#### 优化3：阿里云分片上传改为并发
```java
// 当前：串行上传
while ((bytesRead = inputStream.read(buffer)) != -1) {
    ossClient.uploadPart(uploadPartRequest); // 串行
}

// 优化后：并发上传
ExecutorService executor = Executors.newFixedThreadPool(4); // 4个并发线程
List<Future<PartETag>> futures = new ArrayList<>();
while ((bytesRead = inputStream.read(buffer)) != -1) {
    Future<PartETag> future = executor.submit(() -> {
        return ossClient.uploadPart(uploadPartRequest);
    });
    futures.add(future);
}
// 等待所有分片上传完成
for (Future<PartETag> future : futures) {
    partETags.add(future.get());
}
```
**预期效果**：10个分片并发上传，总时间 ≈ 单分片时间，速度提升约10倍

#### 优化4：优化分片大小
```java
// 当前：固定10MB
// 优化：根据文件大小动态调整
long chunkSize;
if (fileSize < 100L * 1024 * 1024) {
    chunkSize = 20L * 1024 * 1024; // 100MB以下用20MB分片
} else if (fileSize < 500L * 1024 * 1024) {
    chunkSize = 50L * 1024 * 1024; // 500MB以下用50MB分片
} else {
    chunkSize = 100L * 1024 * 1024; // 500MB以上用100MB分片
}
```
**预期效果**：减少分片数量，降低网络请求开销，提升上传速度10%-20%

### 2.2 OSS服务升级建议（需确认）

#### 七牛云OSS
1. **确认当前套餐**：
   - 免费版 → 标准版：可能提升带宽
   - 标准版 → 专业版：更高带宽和性能
2. **联系客服**：
   - 询问当前套餐的实际上传带宽限制
   - 询问升级后的带宽提升
   - 询问是否有CDN加速服务

#### 阿里云OSS
1. **检查账号等级**：
   - 普通账号 vs 企业账号：企业账号可能有更高带宽
2. **地域选择**：
   - 确保OSS地域与服务器地域一致或相近
   - 华东1（杭州）有20Gbps外网上传带宽，比其他地域高
3. **传输加速**：
   - 启用OSS传输加速服务，通过全球加速节点优化路径
   - 可能产生额外费用

### 2.3 网络优化

1. **服务器带宽**：
   - 检查服务器出口带宽是否足够
   - 100MB文件，如果上传时间60秒，需要约13Mbps带宽
   - 如果上传时间30秒，需要约26Mbps带宽

2. **地域选择**：
   - 服务器与OSS在同一地域：使用内网地址（更快，免费）
   - 跨地域：选择距离最近的地域
