### paper_code格式说明

**格式**：`{year}{month}_{province_code}_{paper_type}_{custom_name}`

**示例**：
- `202312_hebei_junior_high_school_english_listening_模拟一`
- `202406_jiangsu_high_school_english_listening`（无自定义名称）

**分组键**（用于OSS路径分组）：
- 格式：`{year}{month}_{province_code}_{paper_type}`
- 示例：`202312_hebei_junior_high_school_english_listening`

### paper_name格式说明

**格式**：`{year}年{month}月{province}{paper_type_label}{custom}`

**示例**：
- `2023年12月河北中考英语听力模拟一`
- `2024年6月江苏高考英语听力`

### OSS路径格式

**新格式**：`paper_packages/{year}{month}_{province}_{paper_type}/{paper_code}_v{version}.zip`

**示例**：
- `paper_packages/202312_hebei_junior_high_school_english_listening/202312_hebei_junior_high_school_english_listening_模拟一_v1.zip`

**向后兼容**：
- 如果无法解析paper_code，使用旧格式：`paper_packages/{paper_type}/{paper_code}_v{version}.zip`
- 如果paper_type也为空，使用：`paper_packages/{paper_code}_v{version}.zip`

### 唯一性约束

- **唯一约束**：通过`paper_code`的UNIQUE约束实现
- **唯一性规则**：年月 + 省份 + 试卷类型 + 自定义名称（如果有）
- **冲突处理**：如果相同年月、省份、类型的试卷已存在，提示使用自定义名称区分

### 注意事项

1. **向后兼容**：
   - 旧的paper_code格式（如`PAPER_YYYYMMDD_序号`）仍然可以正常使用
   - 如果无法解析paper_code，系统会降级到旧格式的OSS路径

2. **数据迁移**：
   - 现有试卷的paper_code不需要修改
   - 新创建的试卷会使用新格式的paper_code
   - 如果编辑现有试卷，可以保持旧格式或更新为新格式

3. **字典数据**：
   - 必须执行`sql/dict_paper_month_province.sql`，否则前端无法选择月份和省份

4. **试卷名称**：
   - 试卷名称由后端自动生成，前端不需要填写
   - 编辑页面中试卷名称显示为只读
