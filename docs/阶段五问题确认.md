# 阶段五问题确认

## ✅ 最新更新：业务数据同步配置方案（已实现）

### 配置方案概述

**设计目标**：
1. **默认使用ZIP包方案**（性能最优）
2. **通过配置开关控制**是否允许业务数据表同步（备选方案）
3. **避免大数据量性能问题**，强制使用ZIP包方案

### 配置位置

配置文件：`{userData}/config.json`

- **macOS**: `~/Library/Application Support/exam-client/config.json`
- **Windows**: `C:\Users\{用户名}\AppData\Roaming\exam-client\config.json`
- **Linux**: `~/.config/exam-client/config.json`

### 配置项

```json
{
  "enableTableSync": false,  // 默认关闭，只使用ZIP包方案
  "maxQuestionsForTableSync": 1000  // 如果开启表同步，最大题目数量阈值
}
```

### 工作流程

1. **默认配置（`enableTableSync: false`）**：
   - 检查ZIP包是否存在
   - 如果有ZIP包 → 使用ZIP包方案（推荐）
   - 如果没有ZIP包 → **提示错误**，要求管理员生成试卷包

2. **开启表同步（`enableTableSync: true`）**：
   - 检查ZIP包是否存在
   - 如果有ZIP包 → 使用ZIP包方案（推荐）
   - 如果没有ZIP包 → 使用业务数据表同步（⚠️ 性能警告）

### 错误提示

**错误码：`NO_ZIP_PACKAGE_AND_TABLE_SYNC_DISABLED`**
```
未检测到ZIP包，且业务数据表同步已禁用。

解决方案：
1. 请在后台管理-试卷管理中生成试卷包
2. 或者联系开发管理者开启业务数据表同步功能（仅适用于小数据量场景）
```

### 实现文件

- ✅ `exam-client/src/database/config.js` - 配置管理类
- ✅ `exam-client/src/database/syncService.js` - 同步逻辑（已更新）

## 原始问题确认

## 一、问题清单

### 1. ✅ 保留业务数据表同步有没有影响？

**答案：没有负面影响，只有好处**

**影响分析**：
- ✅ **兼容性**：如果后端还没有生成ZIP包，客户端仍然可以工作
- ✅ **容错性**：如果ZIP包下载失败，可以降级使用表同步
- ✅ **性能影响**：只有在没有ZIP包时才会同步，有ZIP包时会跳过
- ✅ **代码复杂度**：增加了一些代码，但逻辑清晰

**建议**：**保留**，作为备选方案

---

### 2. ✅ 后端是否已可生成ZIP包？

**答案：是的，后端已经可以生成ZIP包**

**检查结果**：
- ✅ 后端有 `generatePaperPackage()` 方法
- ✅ 生成ZIP包包含：
  - `manifest.json` - 试卷元数据
  - `questions.json` - 题目数据
  - `media/` - 媒体文件目录
- ✅ 生成后上传到OSS
- ✅ 更新 `paper` 表的 `package_hash`、`package_size`、`version` 字段

**生成路径**：
- OSS路径：`paper_packages/{paper_code}_v{version}.zip`
- 这是正确的，不需要调整（OSS是跨平台的）

---

### 3. ⚠️ ZIP包目录不存在，是否需要调整后端生成路径？

**答案：不需要调整后端，但需要实现客户端下载功能**

**说明**：
- 后端生成ZIP包后上传到OSS（这是正确的）
- 客户端需要从OSS下载ZIP包到本地
- 客户端本地路径：`{userData}/paper_packages/{paper_code}_v{version}.zip`
- 这个路径是客户端自己管理的，不需要后端调整

**当前状态**：
- ❌ 客户端还没有实现ZIP包下载功能（阶段六的任务）
- ✅ 表结构已经创建好了（阶段五已完成）

---

### 4. ✅ MySQL中没有paper_package表是否正确？

**答案：完全正确！**

**设计说明**：
- **MySQL（后端）**：
  - ❌ 不需要 `paper_package` 表
  - ✅ ZIP包存储在OSS（对象存储）
  - ✅ `paper` 表存储ZIP包元数据（hash、size、version、URL）

- **SQLite（客户端）**：
  - ✅ 需要 `paper_package` 表
  - ✅ 存储从OSS下载的ZIP包（BLOB或文件路径）
  - ✅ 用于离线使用

**数据流**：
```
后端生成ZIP包 → 上传到OSS → 客户端从OSS下载 → 存储到SQLite的paper_package表
```

---

### 5. ✅ SQLite中的paper_package表是否通过ZIP文件导入？

**答案：是的，但需要实现下载功能**

**流程**：
1. 客户端从OSS下载ZIP包
2. 根据文件大小选择存储方式：
   - <50MB → 存储为BLOB到 `paper_package.package_data`
   - 50-200MB → 存储到文件系统，路径存到 `paper_package.package_path`
   - >200MB → 分片下载，存储到文件系统
3. 保存元数据到 `paper_package` 表

**当前状态**：
- ✅ 表结构已创建
- ❌ 下载功能未实现（阶段六）

---

### 6. ⚠️ 如果不存在ZIP呢？

**答案：使用备选方案（业务数据表同步）**

**逻辑**：
```javascript
if (有ZIP包) {
  // 优先方案：使用ZIP包
  下载ZIP包 → 解压 → 使用
} else {
  // 备选方案：同步业务数据表
  同步 paper、question、question_category 等表
}
```

**当前实现**：
- ✅ `shouldSyncBusinessData()` 会检查是否有ZIP包
- ✅ 如果没有ZIP包，会同步业务数据表
- ✅ 如果有ZIP包，会跳过业务数据表同步
