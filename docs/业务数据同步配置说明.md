# 业务数据同步配置说明

## 一、配置方案概述

### 设计目标

1. **默认使用ZIP包方案**（性能最优）
2. **通过配置开关控制**是否允许业务数据表同步（备选方案）
3. **避免大数据量性能问题**，强制使用ZIP包方案

### 配置位置

配置文件：`{userData}/config.json`

- **macOS**: `~/Library/Application Support/exam-client/config.json`
- **Windows**: `C:\Users\{用户名}\AppData\Roaming\exam-client\config.json`
- **Linux**: `~/.config/exam-client/config.json`

---

## 二、配置项说明

### 默认配置

```json
{
  "enableTableSync": false,
  "maxQuestionsForTableSync": 1000
}
```

### 配置项详解

#### 1. `enableTableSync`（业务数据表同步开关）

- **类型**: `boolean`
- **默认值**: `false`
- **说明**: 
  - `false`: **只使用ZIP包方案**，如果没有ZIP包则提示错误
  - `true`: 如果没有ZIP包，允许使用业务数据表同步（性能较差，仅用于小数据量场景）

#### 2. `maxQuestionsForTableSync`（最大题目数量阈值）

- **类型**: `number`
- **默认值**: `1000`
- **说明**: 如果启用表同步，当题目数量超过此阈值时，会提示使用ZIP包方案

## 四、如何修改配置

### 方法1：手动编辑配置文件

1. 找到配置文件位置（见上方）
2. 编辑 `config.json`：

```json
{
  "enableTableSync": true,
  "maxQuestionsForTableSync": 1000
}
```

3. 重启客户端应用

### 方法2：通过代码修改（开发时）

```javascript
const { getConfig } = require('./database/config')

// 开启业务数据表同步
const config = getConfig()
config.set('enableTableSync', true)
config.saveConfig()
```

## 五、错误提示说明

### 错误码：`NO_ZIP_PACKAGE_AND_TABLE_SYNC_DISABLED`

**触发条件**：
- 未检测到ZIP包
- `enableTableSync: false`（默认配置）

**错误信息**：
```
未检测到ZIP包，且业务数据表同步已禁用。

解决方案：
1. 请在后台管理-试卷管理中生成试卷包
2. 或者联系开发管理者开启业务数据表同步功能（仅适用于小数据量场景）
```

### 错误码：`TABLE_SYNC_TIMEOUT`

**触发条件**：
- 启用了表同步（`enableTableSync: true`）
- 同步过程中超时（可能是数据量过大）

**错误信息**：
```
业务数据同步超时，可能是数据量过大。

解决方案：
1. 请在后台管理-试卷管理中生成试卷包（推荐）
2. 或者联系开发管理者调整配置
```

## 六、性能对比

### ZIP包方案（推荐）

| 数据量 | 同步时间 | 网络流量 |
|--------|----------|----------|
| 1,000题 | ~10秒 | ~50MB |
| 10,000题 | ~30秒 | ~200MB |
| 100,000题 | ~2分钟 | ~1GB |

### 业务数据表同步（备选）

| 数据量 | 同步时间 | 网络流量 |
|--------|----------|----------|
| 100题 | ~5秒 | ~1MB |
| 1,000题 | ~30秒 | ~10MB |
| 10,000题 | ~30分钟 | ~100MB |
| 100,000题 | ~5小时 | ~1GB |

**结论**：ZIP包方案在大数据量场景下性能优势明显。

## 七、最佳实践

### ✅ 推荐做法

1. **默认配置**：保持 `enableTableSync: false`
2. **生成ZIP包**：在后台管理-试卷管理中为所有试卷生成ZIP包
3. **使用ZIP包方案**：客户端自动检测并使用ZIP包

### ⚠️ 特殊情况

1. **小数据量测试**：可以临时开启 `enableTableSync: true`
2. **开发调试**：可以临时开启表同步，方便调试
3. **生产环境**：**强烈建议使用ZIP包方案**

## 九、总结

### 核心原则

1. **默认使用ZIP包方案**（性能最优）
2. **配置开关控制备选方案**（仅用于特殊情况）
3. **明确提示性能风险**（避免误用）

### 适用场景

| 场景 | 推荐配置 |
|------|----------|
| **生产环境** | `enableTableSync: false`（默认） |
| **小数据量测试** | `enableTableSync: true`（临时） |
| **开发调试** | `enableTableSync: true`（临时） |
| **大数据量场景** | `enableTableSync: false`（必须） |

## 十、常见问题

### Q1: 为什么默认关闭表同步？

**A**: 业务数据表同步在大数据量场景下性能极差（可能需要数小时），默认关闭可以避免性能问题。

### Q2: 什么时候可以开启表同步？

**A**: 仅适用于小数据量场景（<1000题），且仅用于测试或开发调试。

### Q3: 如何知道数据量是否过大？

**A**: 如果同步时间超过5分钟，建议使用ZIP包方案。

### Q4: 配置文件在哪里？

**A**: 见上方"配置位置"章节，不同操作系统路径不同。

### Q5: 修改配置后需要重启吗？

**A**: 是的，需要重启客户端应用才能生效。
