# 试卷类型功能实现总结

## 一、功能概述

为试卷管理添加"试卷类型"字段，实现：
1. 试卷包根据试卷类型区分存储路径
2. 学员获取ZIP包时，根据学员档案的适用试卷类型自动过滤

## 二、数据库变更

### MySQL（后端）

**DDL语句**：`sql/add_paper_type_column.sql`

```sql
-- 1. 为paper表添加paper_type字段
ALTER TABLE `paper` 
ADD COLUMN `paper_type` VARCHAR(50) COMMENT '试卷类型（字典编码：paper_type）' 
AFTER `paper_code`;

-- 2. 添加索引
ALTER TABLE `paper` 
ADD INDEX `idx_paper_type` (`paper_type`);

ALTER TABLE `paper` 
ADD INDEX `idx_paper_type_status` (`paper_type`, `status`);
```

### SQLite（客户端）

**自动更新**：`exam-client/src/database/db.js`
- 在`initPaperTables`中添加`paper_type TEXT`字段
- 添加`addColumnIfNotExists`方法，用于向后兼容
- 在`ensurePaperTablesExist`中自动检查并添加字段

## 六、工作流程

### 1. 试卷包生成流程

```
后台管理 → 生成试卷包
    ↓
检查试卷类型
    ↓
┌───┴───┐
│       │
有类型   无类型（向后兼容）
│       │
↓       ↓
paper_packages/    paper_packages/
{paper_type}/      {paper_code}_v{version}.zip
{paper_code}_v{version}.zip
```

### 2. 学员获取试卷包流程

```
学员登录 → 获取试卷列表
    ↓
获取学员档案（适用试卷类型）
    ↓
过滤试卷列表（只返回适用试卷类型的试卷）
    ↓
选择试卷 → 下载试卷包
    ↓
验证权限（试卷类型是否匹配）
    ↓
下载ZIP包
```

## 七、权限控制

### 规则

1. **试卷列表查询**：
   - 如果学员档案有适用试卷类型，自动过滤
   - 如果学员有多个适用试卷类型，可以分别查询
   - 如果学员尝试查询非适用试卷类型，返回403错误

2. **试卷包下载**：
   - 下载前验证试卷类型是否在学员档案的适用试卷类型中
   - 如果不匹配，抛出异常，拒绝下载
