## 技术实现细节

### 数据获取方式

- **优先使用本地JSON数据**: `manifest.json`, `questions.json`
- **数据来源**: `paperService.js` 中的 `extractPaperPackage` 或 `extractQuickStartPackage`
- **IPC通信**: 通过 `ipcRenderer.invoke('paper:getQuestions')` 等获取数据
- **中文编码处理**: 所有文本字段使用 `String()` 显式转换

### 音频播放机制

- 使用 HTML5 `<audio>` 元素
- 支持 Base64 和 `file://` URL
- 音频路径处理：支持 `media/`, `sections/`, `volumes/` 等目录结构
- 音频可视化：使用 Web Audio API 和 Canvas 绘制波浪线

### 样式规范

- 大题标题：固定顶部，左边距 60px，字体 18px，蓝色 (#409EFF)
- 题目内容：可滚动区域，左边距 60px
- 分页按钮：固定底部，左边距 60px
- 学生信息：右侧面板，蓝色加粗显示

### 数据库相关

- **数据库文件**: `exam-client/src/database/db.js`
- **试卷服务**: `exam-client/src/database/paperService.js`
- **登录服务**: `exam-client/src/database/loginService.js`
- **数据库类型**: SQLite (better-sqlite3)

## 注意事项

1. **中文编码**: 所有文本字段必须使用 `String()` 显式转换，优先使用本地JSON数据
2. **音频路径**: 从 `manifest.json` 的 `sections` 中直接获取 `instructionAudioPath`
3. **样式一致性**: 保持与 Broadcast.vue 相同的布局和样式
4. **离线支持**: 所有数据操作需要考虑离线环境
5. **错误处理**: 音频加载失败、数据获取失败等需要优雅降级
