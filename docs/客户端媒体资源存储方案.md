# 客户端媒体资源存储方案

## 一、需求分析

### 1.1 使用场景

- **主要场景**：离线使用（大部分时间）
- **次要场景**：在线使用（同步数据时）
- **媒体类型**：图片、音频、视频
- **存储位置**：客户端本地

### 1.2 关键问题

1. **离线访问**：必须支持离线访问媒体文件
2. **存储空间**：需要管理磁盘空间
3. **同步效率**：试卷包（ZIP）已包含媒体文件
4. **访问性能**：本地文件访问速度快

---

## 二、方案对比

### 方案A：OSS方案（不推荐）

**思路**：媒体文件存储在OSS，客户端通过URL访问

**优点**：
- ✅ 统一管理
- ✅ 不占用客户端磁盘空间
- ✅ 在线时访问方便

**缺点**：
- ❌ **离线时无法访问**（主要问题）
- ❌ 需要网络连接
- ❌ 不适合离线答题场景

**结论**：❌ **不适合**，因为客户端主要是离线使用

---

### 方案B：本地磁盘文件方案（推荐）

**思路**：媒体文件存储在客户端本地磁盘，从试卷包（ZIP）解压后存储

**优点**：
- ✅ **支持离线访问**（核心优势）
- ✅ 访问速度快（本地文件）
- ✅ 试卷包已包含媒体文件，无需额外下载
- ✅ 不依赖网络

**缺点**：
- ⚠️ 占用磁盘空间（需要管理）
- ⚠️ 需要管理文件生命周期

**结论**：✅ **推荐**，最适合离线使用场景

---

### 方案C：混合方案（可选）

**思路**：在线时从OSS加载，离线时从本地加载

**优点**：
- ✅ 在线时节省本地空间
- ✅ 离线时可用

**缺点**：
- ⚠️ 实现复杂
- ⚠️ 需要判断在线/离线状态
- ⚠️ 需要管理两套存储

**结论**：⚠️ **可选**，但增加了复杂度

## 三、推荐方案：本地磁盘文件方案

### 3.1 存储结构

```
应用数据目录/
├── exam.db (SQLite数据库)
├── paper_packages/          # 试卷包存储目录
│   ├── {paper_code}_v{version}.zip
│   └── {paper_code}_v{version}.zip.part1 (分片)
└── media/                    # 媒体文件存储目录（解压后）
    ├── {paper_code}/         # 按试卷分组
    │   ├── q_{question_id}/  # 按题目分组
    │   │   ├── title.jpg     # 题目媒体
    │   │   └── options/      # 选项媒体
    │   │       ├── a.mp3
    │   │       └── b.mp3
    │   └── q_{question_id}/
    │       └── video.mp4
    └── intro/                 # 试卷独白音频
        └── {paper_code}.mp3
```

### 3.2 工作流程

```
1. 同步试卷包（ZIP）
   ↓
2. 解压试卷包到临时目录
   ↓
3. 读取 manifest.json 和 questions.json
   ↓
4. 将媒体文件移动到 media/{paper_code}/ 目录
   ↓
5. 在SQLite中存储媒体文件路径（相对路径）
   ↓
6. 答题时从本地路径加载媒体文件
```

### 3.3 数据库设计调整

**SQLite表结构**（优化版）：

```sql
-- 试卷包表（存储ZIP包）
CREATE TABLE `paper_package` (
  `id` INTEGER PRIMARY KEY AUTOINCREMENT,
  `paper_id` INTEGER NOT NULL UNIQUE,
  `paper_code` TEXT NOT NULL UNIQUE,
  `package_data` BLOB,  -- 小文件直接存储
  `package_path` TEXT,  -- 大文件存储路径
  `package_hash` TEXT NOT NULL,
  `package_size` INTEGER NOT NULL,
  `storage_type` INTEGER DEFAULT 0,
  `version` INTEGER DEFAULT 1,
  `sync_time` INTEGER NOT NULL,
  `is_active` INTEGER DEFAULT 1
);

-- 试卷索引表（存储试卷元数据）
CREATE TABLE `paper_index` (
  `id` INTEGER PRIMARY KEY AUTOINCREMENT,
  `paper_id` INTEGER NOT NULL UNIQUE,
  `paper_code` TEXT NOT NULL,
  `paper_name` TEXT NOT NULL,
  `total_questions` INTEGER DEFAULT 0,
  `total_score` REAL DEFAULT 0,
  `duration` INTEGER,
  `intro_audio_url` TEXT,  -- 在线URL（可选）
  `intro_audio_path` TEXT, -- 本地路径（media/intro/{paper_code}.mp3）
  `intro_audio_duration` INTEGER,
  `version` INTEGER DEFAULT 1,
  `package_hash` TEXT,
  `sync_time` INTEGER,
  `is_active` INTEGER DEFAULT 1
);

-- 题目媒体文件索引表（新增，用于快速查找媒体文件）
CREATE TABLE `question_media_index` (
  `id` INTEGER PRIMARY KEY AUTOINCREMENT,
  `paper_id` INTEGER NOT NULL,
  `question_id` INTEGER NOT NULL,
  `media_type` INTEGER NOT NULL COMMENT '1-题目媒体，2-选项媒体',
  `option_id` INTEGER,
  `blank_area_id` INTEGER,
  `media_name` TEXT NOT NULL,
  `media_path` TEXT NOT NULL COMMENT '本地文件路径（相对路径）',
  `media_format` TEXT,
  `media_size` INTEGER,
  `media_duration` INTEGER,
  INDEX `idx_paper_question` (`paper_id`, `question_id`)
);
```

### 3.4 文件路径设计

**相对路径方案**（推荐）：
- 存储相对路径：`media/{paper_code}/q_{question_id}/title.jpg`
- 访问时拼接：`{appDataPath}/media/{paper_code}/q_{question_id}/title.jpg`

**优点**：
- ✅ 数据库可移植（路径不包含绝对路径）
- ✅ 易于管理
- ✅ 支持应用数据目录迁移

## 四、实现方案

### 4.1 试卷包解压流程

```javascript
// exam-client/src/database/paperService.js

class PaperService {
  constructor(db) {
    this.db = db.getDB()
    this.appDataPath = app.getPath('userData')
    this.mediaBasePath = path.join(this.appDataPath, 'media')
    this.packageBasePath = path.join(this.appDataPath, 'paper_packages')
  }

  /**
   * 解压试卷包并提取媒体文件
   */
  async extractPaperPackage(paperCode) {
    // 1. 从SQLite或文件系统读取ZIP包
    const zipData = await this.getPaperPackage(paperCode)
    
    // 2. 解压到临时目录
    const tempDir = path.join(this.appDataPath, 'temp', paperCode)
    await this.extractZip(zipData, tempDir)
    
    // 3. 读取manifest.json和questions.json
    const manifest = JSON.parse(fs.readFileSync(path.join(tempDir, 'manifest.json')))
    const questions = JSON.parse(fs.readFileSync(path.join(tempDir, 'questions.json')))
    
    // 4. 移动媒体文件到media目录
    const mediaDir = path.join(this.mediaBasePath, paperCode)
    await this.moveMediaFiles(tempDir, mediaDir, questions)
    
    // 5. 保存媒体文件索引到SQLite
    await this.saveMediaIndex(paperCode, questions, mediaDir)
    
    // 6. 清理临时目录
    await this.cleanupTempDir(tempDir)
    
    return { manifest, questions, mediaDir }
  }

  /**
   * 移动媒体文件到media目录
   */
  async moveMediaFiles(tempDir, mediaDir, questions) {
    // 确保media目录存在
    if (!fs.existsSync(mediaDir)) {
      fs.mkdirSync(mediaDir, { recursive: true })
    }
    
    for (const question of questions) {
      const questionMediaDir = path.join(mediaDir, `q_${question.questionId}`)
      if (!fs.existsSync(questionMediaDir)) {
        fs.mkdirSync(questionMediaDir, { recursive: true })
      }
      
      // 移动题目媒体
      if (question.mediaFiles) {
        for (const mediaFile of question.mediaFiles) {
          const srcPath = path.join(tempDir, 'media', `q_${question.questionId}`, mediaFile)
          const destPath = path.join(questionMediaDir, mediaFile)
          
          if (fs.existsSync(srcPath)) {
            fs.copyFileSync(srcPath, destPath) // 复制而不是移动，保留ZIP中的文件
          }
        }
      }
    }
  }

  /**
   * 保存媒体文件索引到SQLite
   */
  async saveMediaIndex(paperCode, questions, mediaDir) {
    const stmt = this.db.prepare(`
      INSERT OR REPLACE INTO question_media_index 
      (paper_id, question_id, media_type, option_id, media_name, media_path, media_format, media_size)
      VALUES (?, ?, ?, ?, ?, ?, ?, ?)
    `)
    
    for (const question of questions) {
      // 保存题目媒体
      if (question.titleMedia) {
        const relativePath = `media/${paperCode}/q_${question.questionId}/${question.titleMedia.name}`
        stmt.run(
          question.paperId,
          question.questionId,
          1, // 题目媒体
          null,
          question.titleMedia.name,
          relativePath,
          question.titleMedia.format,
          question.titleMedia.size
        )
      }
      
      // 保存选项媒体
      if (question.options) {
        for (const option of question.options) {
          if (option.media) {
            const relativePath = `media/${paperCode}/q_${question.questionId}/options/${option.media.name}`
            stmt.run(
              question.paperId,
              question.questionId,
              2, // 选项媒体
              option.id,
              option.media.name,
              relativePath,
              option.media.format,
              option.media.size
            )
          }
        }
      }
    }
  }

  /**
   * 获取媒体文件绝对路径
   */
  getMediaAbsolutePath(relativePath) {
    return path.join(this.appDataPath, relativePath)
  }

  /**
   * 检查媒体文件是否存在
   */
  mediaFileExists(relativePath) {
    const absolutePath = this.getMediaAbsolutePath(relativePath)
    return fs.existsSync(absolutePath)
  }
}
```

## 五、存储空间管理

### 5.1 清理策略

```javascript
/**
 * 清理旧版本试卷包和媒体文件
 */
async cleanupOldVersions(retentionDays = 90) {
  const cutoffTime = Date.now() - (retentionDays * 24 * 60 * 60 * 1000)
  
  // 1. 查询需要清理的试卷包
  const oldPackages = this.db.prepare(`
    SELECT paper_code, version FROM paper_package
    WHERE sync_time < ? AND is_active = 0
  `).all(cutoffTime)
  
  // 2. 删除旧版本媒体文件
  for (const pkg of oldPackages) {
    const mediaDir = path.join(this.mediaBasePath, pkg.paper_code)
    if (fs.existsSync(mediaDir)) {
      fs.rmSync(mediaDir, { recursive: true, force: true })
    }
  }
  
  // 3. 删除旧版本ZIP包
  for (const pkg of oldPackages) {
    if (pkg.package_path) {
      if (fs.existsSync(pkg.package_path)) {
        fs.unlinkSync(pkg.package_path)
      }
    }
  }
  
  // 4. 从数据库删除记录
  this.db.prepare(`
    DELETE FROM paper_package WHERE sync_time < ? AND is_active = 0
  `).run(cutoffTime)
}
```

## 七、推荐方案确认

### 7.1 最终推荐：本地磁盘文件方案

**理由**：
1. ✅ **离线支持**：完全支持离线访问（核心需求）
2. ✅ **性能优秀**：本地文件访问速度快
3. ✅ **实现简单**：试卷包解压后直接使用
4. ✅ **无需OSS**：不依赖外部服务

### 7.2 存储结构

```
应用数据目录/
├── exam.db
├── paper_packages/     # ZIP包存储
└── media/              # 媒体文件（解压后）
    └── {paper_code}/
        └── q_{question_id}/
```

### 7.3 关键设计

1. **相对路径存储**：数据库存储相对路径，访问时拼接绝对路径
2. **按试卷分组**：媒体文件按试卷分组，便于管理
3. **按题目分组**：每个题目的媒体文件独立目录
4. **定期清理**：清理旧版本媒体文件，释放空间
