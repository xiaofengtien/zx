# 数据量性能分析

## 一、数据量估算

### 假设场景
- **题目数量**：100,000 条
- **试卷数量**：5,000 条
- **题目答案**：平均每题4个选项 = 400,000 条
- **完形填空区域**：假设10%题目是完形填空 = 10,000 条
- **题目媒体文件**：假设30%题目有媒体 = 30,000 条

### 总数据量
- **题目相关**：100,000 + 400,000 + 10,000 + 30,000 = **540,000 条记录**
- **试卷相关**：5,000 条
- **试卷-题目关联**：假设每份试卷50题 = 250,000 条
- **总计**：约 **795,000 条记录**

## 五、结论

### ⚠️ 重要结论

1. **业务数据表同步不适合大数据量场景**：
   - 数据量大时，同步时间过长（1.5-3小时）
   - API调用次数过多（10万+次）
   - 严重影响用户体验

2. **ZIP包方案是唯一可行的方案**：
   - 同步时间短（1-2分钟）
   - API调用次数少（1次）
   - 性能优势明显（75-150倍提升）

3. **业务数据表同步只能作为临时备选**：
   - 仅适用于数据量很小的情况（<1000条题目）
   - 或者后端还没有生成ZIP包时的临时方案
