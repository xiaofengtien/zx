# 答题流程完整需求设计

## 一、需求概述

本文档详细描述了考试答题流程的完整需求，包括题目显示、答题流程控制、卷别切换、答题完成等各个环节的具体实现要求。

## 二、题目显示区域设计

### 2.1 显示位置
- **页面**：大题页面（`SectionList.vue`）
- **固定显示**：大题标题固定在页面顶部
- **题目数量**：当前页面显示 **3-5题**
- **选项布局**：选项一行两个排列

### 2.2 布局要求
- 大题标题：固定在顶部，左边距60px
- 题目内容区域：可滚动，显示3-5题
- 选项排列：每行显示2个选项
- 右侧学生信息区域：显示题目序号列表，需要标注已答题状态

## 三、答题流程详细逻辑

### 3.1 初始进入大题页面

1. **播放答题音频**
   - 播放完答题音频（"我们要开始答题了"）
   - 播放完成后等待 **3秒**

2. **显示题目并播放第一题音频**
   - 3秒后显示题目
   - **同时**播放第一题的音频
   - **光标闪烁**在当前题目（用作提示）
   - 题目显示区域显示3-5题（当前题目高亮）

3. **第一题答题时间**
   - 第一题音频播放完成后
   - 停留时间：`paper_section.answer_time`（单位：秒）
   - 学生在此时间内答题

4. **自动播放下一题**
   - 当 `answer_time` 时间到了
   - **自动播放**下一题的音频
   - 光标闪烁移动到下一题
   - 上一题进入"已过期"状态（不允许修改）

5. **重复流程**
   - 下一题音频播放完成后
   - 停留 `answer_time` 秒
   - 时间到了自动播放再下一题
   - 重复直到最后一题完成

6. **最后一题完成**
   - 最后一题音频播放完成
   - 停留 `answer_time` 秒
   - 时间到了，停止自动播放
   - 跳转到当前卷别作答完成提示页面

### 3.2 答题状态管理

#### 3.2.1 题目状态
- **未开始**：题目还未显示
- **当前题目**：正在答题的题目（光标闪烁）
- **已答题**：学生已作答，但还在答题时间内（可以修改）
- **已过期**：答题时间已过，不允许修改（即使已作答或未作答）

#### 3.2.2 答题限制规则
- **答题过程中**：答题结果不允许修改（一旦跳转到下一题，上一题就锁定）
- **时间过期**：如果已经跳转到下一题了，作答时间到了之后，已经过期的不管作答还是没作答都不允许修改了
- **右侧题目列表**：需要标注已答题状态（已答题的题目显示不同颜色或标记）

## 四、卷别切换流程

### 4.1 当前卷别答题完成

1. **跳转到卷别完成提示页面**
   - 显示："X卷作答完成"
   - 检查是否有下一个卷别

2. **判断是否有下一卷别**
   - **如果没有下一卷别**：
     - 显示："继续作答下一个卷别的提示"（如果有的话）
     - 或者直接跳转到答题完成页面
   
   - **如果有下一卷别**：
     - 检查是否有中场设置（`paper_intermission`）
     - 当前页面停留 **3秒**

### 4.2 中场处理（如果有中场设置）

1. **跳转到中场页面**
   - 3秒后自动跳转到中场页面
   - **自动播放**中场音频
   - **居中显示**中场文本（可以完全复制注意事项页面的样式）

2. **中场音频播放完成**
   - 音频播放完成后
   - 停留 **3秒**

3. **跳转到卷别播报页面**
   - 3秒后跳转到试卷卷别播报页面（`Broadcast.vue`）
   - 显示下一个卷别名称
   - 播放下一个卷别的音频

### 4.3 进入下一卷别

1. **卷别播报完成**
   - 卷别音频播放完成后
   - 停留3秒
   - 跳转到大题列表页面（`SectionList.vue`）

2. **重复答题流程**
   - 重复之前的答题逻辑
   - 播放大题音频
   - 显示题目
   - 自动播放题目音频
   - 答题时间控制
   - 直到最后一题完成

### 4.4 所有卷别完成

1. **最后一卷答题完成**
   - 跳转到答题完成页面
   - 结束答题

2. **答题完成页面功能**
   - 根据后台试卷设置判断是否允许回放检查
   - 如果允许：显示"回放检查"按钮
   - 如果不允许：直接显示完成信息

## 五、数据保存与同步

### 5.1 答题结果保存

1. **本地保存**
   - 答题过程中实时保存到 SQLite 本地数据库
   - 保存表：`app_user_paper_question_result`
   - 保存字段：
     - `paper_id`：试卷ID
     - `app_user_id`：用户ID
     - `question_id`：题目ID
     - `answer_ids`：答案ID（多个用逗号分隔）
     - `user_answer`：用户答案文本
     - `result`：结果（0-错误，1-正确）
     - `question_sort`：题目序号
     - `time_spent`：用时（秒）
     - `answer_time`：答题时间戳
     - `sync_status`：同步状态（0-未同步，1-已同步）

2. **等待联网后上传**
   - 答题完成后，标记所有答题结果为 `sync_status = 0`
   - 等待联网后，自动上传到后台
   - 上传成功后，更新 `sync_status = 1`

### 5.2 试卷数据修改

1. **答题次数减少**
   - 答题完成后，试卷的答题次数需要减少
   - 更新 `app_user_paper_info` 表
   - 字段：`attempt_count` 或类似字段

2. **答题记录状态**
   - 更新 `app_user_paper_info.status` 为"已完成"
   - 记录完成时间

## 六、页面流转图

```
Broadcast.vue (卷别播报)
    ↓ (音频播放完成 + 3秒)
SectionList.vue (大题列表/题目显示)
    ↓ (大题音频播放完成)
题目显示模式
    ↓ (答题音频播放完成 + 3秒)
显示题目 + 播放第一题音频
    ↓ (第一题音频播放完成)
停留 answer_time 秒（答题时间）
    ↓ (时间到)
自动播放下一题音频
    ↓ (下一题音频播放完成)
停留 answer_time 秒（答题时间）
    ↓ (重复直到最后一题)
最后一题完成
    ↓
卷别完成提示页面
    ↓ (检查是否有下一卷别)
    ├─ 没有下一卷别 → 答题完成页面
    └─ 有下一卷别
        ↓ (检查中场设置)
        ├─ 没有中场 → 直接跳转到 Broadcast.vue (下一卷别)
        └─ 有中场
            ↓ (停留3秒)
            中场页面 (播放中场音频 + 显示中场文本)
            ↓ (音频播放完成 + 3秒)
            Broadcast.vue (下一卷别播报)
            ↓ (重复流程)
```

## 七、关键时间节点

| 节点 | 时间 | 说明 |
|------|------|------|
| 答题音频播放完成 | +3秒 | 显示题目 |
| 题目音频播放完成 | +answer_time秒 | 答题时间 |
| 答题时间到 | 自动播放下一题 | 上一题锁定 |
| 卷别完成 | +3秒 | 检查下一卷别/中场 |
| 中场音频播放完成 | +3秒 | 跳转卷别播报 |
| 卷别播报完成 | +3秒 | 跳转大题列表 |

## 八、技术实现要点

### 8.1 题目显示
- 使用 `SectionList.vue` 作为主要页面
- 固定大题标题在顶部
- 题目列表显示3-5题，支持滚动
- 选项一行两个排列（使用 CSS Grid 或 Flexbox）

### 8.2 音频播放控制
- 使用 HTML5 `<audio>` 元素
- 监听 `ended` 事件触发下一步
- 使用定时器控制停留时间
- 音量控制同步到所有音频元素

### 8.3 光标闪烁效果
- 当前题目添加 CSS 动画（闪烁效果）
- 使用 `:class` 动态绑定当前题目
- 题目切换时更新闪烁状态

### 8.4 答题状态管理
- 使用 Vue 响应式数据管理题目状态
- 状态包括：未开始、当前、已答题、已过期
- 右侧题目列表根据状态显示不同样式

### 8.5 时间控制
- 使用 `setTimeout` 控制停留时间
- 使用 `clearTimeout` 清理定时器
- 答题时间使用倒计时显示

### 8.6 数据持久化
- 使用 IPC 调用主进程保存数据
- 答题结果实时保存到 SQLite
- 同步状态标记，等待联网上传
