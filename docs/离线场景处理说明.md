# 离线场景处理说明

## 一、问题分析

### 1. 当前方案的问题

在 `syncPaperPackage` 方法中，我们强制从服务器获取最新的试卷信息（`forceRefresh = true`）：

```javascript
const paper = await this.getPaperInfo(paperCode, token, true)
```

**问题**：
- ✅ **在线场景**：正常工作，确保使用最新的hash
- ❌ **离线场景**：如果网络请求失败，会导致整个同步流程失败
- ❌ **离线场景**：用户无法使用本地已有的ZIP包

### 2. 离线场景的需求

- **离线时**：应该使用本地SQLite中已有的试卷信息
- **离线时**：如果本地已有ZIP包，应该能够正常使用
- **离线时**：不应该因为网络错误而阻止用户使用本地数据

## 二、解决方案

### 1. 降级策略

**策略**：优先从服务器获取最新信息，如果失败则降级到本地缓存

```javascript
// 1. 尝试从服务器获取最新信息
try {
  paper = await this.getPaperInfo(paperCode, token, true)
} catch (error) {
  // 2. 如果是网络错误，降级到本地缓存
  if (isNetworkError(error)) {
    paper = await this.getPaperInfo(paperCode, token, false)
  } else {
    throw error // 其他错误（如认证失败），直接抛出
  }
}
```

### 2. 网络错误判断

**判断条件**：
- `error.message.includes('网络')`
- `error.message.includes('ECONNREFUSED')`
- `error.message.includes('timeout')`
- `error.message.includes('ENOTFOUND')`
- `error.code === 'ECONNREFUSED'`
- `error.code === 'ENOTFOUND'`
- `error.code === 'ETIMEDOUT'`

### 3. 在 `getPaperInfo` 中的降级

**双重降级**：
1. 在 `syncPaperPackage` 中：如果服务器请求失败，降级到本地缓存
2. 在 `getPaperInfo` 中：如果 `forceRefresh = true` 且服务器请求失败，也降级到本地缓存

```javascript
async getPaperInfo(paperCode, token, forceRefresh = false) {
  try {
    if (forceRefresh) {
      // 从服务器获取
      const response = await axios.post(...)
      // ...
    }
  } catch (error) {
    if (forceRefresh) {
      // 降级到本地缓存
      const localPaper = this.db.prepare(...).get(paperCode)
      if (localPaper) {
        return localPaper
      }
    }
    throw error
  }
}
```

## 三、场景分析

### 1. 在线场景（正常）

```
1. 从服务器获取最新试卷信息 → 成功
2. 使用最新的hash下载ZIP包 → 成功
3. hash匹配，下载成功
```

### 2. 离线场景（降级）

```
1. 尝试从服务器获取最新试卷信息 → 失败（网络错误）
2. 降级到使用本地SQLite中的试卷信息 → 成功
3. 检查本地是否已有ZIP包 → 如果有，使用本地ZIP包
4. 如果没有，提示需要联网下载
```

### 3. 混合场景（部分离线）

```
1. 尝试从服务器获取最新试卷信息 → 失败（网络错误）
2. 降级到使用本地SQLite中的试卷信息 → 成功
3. 检查本地ZIP包的hash是否与本地信息一致 → 如果一致，使用本地ZIP包
4. 如果不一致，提示需要联网更新
```

## 四、实现细节

### 1. 错误处理层次

**第一层**：`syncPaperPackage` 中的错误处理
- 捕获网络错误
- 降级到本地缓存

**第二层**：`getPaperInfo` 中的错误处理
- 如果 `forceRefresh = true` 且服务器请求失败
- 降级到本地SQLite查询

### 2. 日志记录

**在线模式**：
```
从服务器获取到的试卷信息: 版本=15, hash=a8f6003e...
```

**离线模式（降级）**：
```
网络错误，降级到使用本地缓存: ECONNREFUSED
使用本地缓存的试卷信息: 版本=14, hash=f9c72f49...
⚠️ 注意：离线模式下使用的是本地缓存的hash，可能与服务器不一致
```

### 3. 用户体验

**在线时**：
- 自动获取最新信息
- 自动更新本地缓存
- 确保hash一致性

**离线时**：
- 使用本地已有数据
- 不阻止用户使用
- 提示可能的数据不一致

## 五、注意事项

### 1. Hash不一致的处理

**离线场景下**：
- 本地缓存的hash可能与服务器不一致
- 如果本地ZIP包的hash与本地缓存的hash一致，可以使用
- 如果本地ZIP包的hash与本地缓存的hash不一致，需要联网更新

### 2. 版本管理

**离线场景下**：
- 使用本地缓存的版本号
- 可能不是最新版本
- 需要联网后才能更新到最新版本

### 3. 数据一致性

**离线场景下**：
- 数据可能不是最新的
- 但可以正常使用
- 联网后会自动更新

## 六、测试场景

### 1. 完全在线
- ✅ 从服务器获取最新信息
- ✅ 使用最新hash下载ZIP包
- ✅ hash匹配，下载成功

### 2. 完全离线
- ✅ 使用本地缓存的试卷信息
- ✅ 如果本地有ZIP包，可以使用
- ✅ 如果本地没有ZIP包，提示需要联网

### 3. 部分离线（网络不稳定）
- ✅ 尝试从服务器获取，失败后降级
- ✅ 使用本地缓存继续工作
- ✅ 不中断用户操作

### 4. 认证失败（非网络错误）
- ✅ 不降级，直接抛出错误
- ✅ 提示用户重新登录

## 七、总结

**优势**：
1. ✅ 在线时确保使用最新信息
2. ✅ 离线时可以使用本地数据
3. ✅ 网络不稳定时自动降级
4. ✅ 不中断用户操作

**注意事项**：
1. ⚠️ 离线时数据可能不是最新的
2. ⚠️ 需要提示用户可能的数据不一致
3. ⚠️ 联网后需要重新同步
